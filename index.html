What is openGH

It’s an open initiative created to be a meeting point for all code enthusiasts inside Grasshopper3d community. If you have been searching for a place where research, share and learn about coding related to Grasshopper with other people around the world, this is it.

How could I contribute?

The first thing you need to start a project is a personal GitHub account. Then, just send and email to openghdev@gmail.com with your project proposal and we will open a new repository (with a small description and README and LICENSE files) giving you full administration permissions. After that you are responsible for managing the repository, granting access and permissions for every new participant. The OpenGH team will be there to help you as far as we can in any trouble you have with the repository administration. Our idea is to make a self-managed community, the coding translation of the official Grasshopper3d forum.

Rather Drive Stick?

If you prefer to not use the automatic generator, push a branch named gh-pages to your repository to create a page manually. In addition to supporting regular HTML content, GitHub Pages support Jekyll, a simple, blog aware static site generator written by our own Tom Preston-Werner. Jekyll makes it easy to create site-wide headers and footers without having to copy them across every page. It also offers intelligent blog support and other advanced templating features.

Authors and Contributors

You can @mention a GitHub username to generate a link to their profile. The resulting <a> element will link to the contributor's GitHub Profile. For example: In 2007, Chris Wanstrath (@defunkt), PJ Hyett (@pjhyett), and Tom Preston-Werner (@mojombo) founded GitHub.

Support or Contact

Having trouble with Pages? Check out the documentation at http://help.github.com/pages or contact support@github.com and we’ll help you sort it out.
